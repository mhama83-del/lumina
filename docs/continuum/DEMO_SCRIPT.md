# Five-Minute Demo Script

Pre-seeded: c01_amina has a Supported dashboard claim but SQL is only *Stated* (the Critical gap).
Application #1 (amina→Nova) is `under_review`. Application #2 (priya→Apex) is deliberately stale.

- **0:00 Problem/position** — `/` then `/how-it-works`. Three promises: evidence not personality;
  candidate-controlled sharing; every application has a next owner + expected update.
- **0:35 Candidate evidence** — `/demo/scenarios/c01_amina` → **My Evidence**. Show Meridian Map
  legend (dashed reflection vs filled evidence) and the text-alternative table. SQL shows *Stated*.
- **1:25 Role Context** — **Role Context** (`/candidate/roles/data-analyst`). RER is explained per
  requirement; SQL Critical is 1/3 with a Question to Confirm. Not a hiring score.
- **2:10 Consent & apply** — **Preview what the employer will see**. The preview *is* the employer
  view (bound by preview hash). Submit with active availability.
- **3:00 Outcome Loop** — switch to `e01_nova` → **Review queue** → open the application →
  **Request clarification**. Switch back to `c01_amina` → the timeline shows the same event, current
  owner and expected update. No ghosting.
- **3:45 Stale accountability** — switch to `t01_operator` → **Control Tower**. Application #2 is a
  stale exception. Operator can **Remind current owner** — but cannot decide the hire.
- **4:25 Ecosystem** — `u01_university` → cohort aggregate + intervention (no individual list).
- **5:00 Close** — "Continuum is the operating layer connecting evidence, human review, accountable
  outcomes and targeted support across Talentbank's ecosystem."

### Judge Q&A quick answers
- *Different from Passport?* Passport is the profile; Continuum is the evidence→outcome layer.
- *Why doesn't AI hire?* Hiring is a human decision; Continuum makes evidence + questions clearer.
- *Is RER a hiring score?* No — role-specific evidence coverage, explained per requirement; consent
  and availability are gates, never merit points.
- *Passport integrated?* Mock adapter + integration-ready contract; real API pending validation.
